# Neovim ヘルプ・ショートカット一覧

## 保存

| キー | 操作 |
|---|---|
| `Cmd+S` | 手動保存 |

※ `autowriteall` が有効なため、バッファ切り替え時などに自動保存される。`Cmd+S` は明示的に保存したい場合に使う。
※ Alacritty 経由で F18 に変換して動作。

## ファイル検索・コードジャンプ

| キー | 操作 |
|---|---|
| `Cmd+P` | ファイル検索 (キャッシュ + fs_event で自動更新) |
| `Space` → `ff` | ファイル検索 (同上) |
| `Cmd+Shift+F` | テキスト検索 (Telescope live_grep) |
| `Space` → `fg` | テキスト検索 (同上) |
| `gd` | 定義へジャンプ（別ファイルも可） |
| `gi` | 実装へジャンプ (implementation) |
| `gr` | 参照を一覧表示 |
| `K` | ホバードキュメント表示 |
| `<F2>` | シンボルをリネーム |
| `Space` → `ca` | コードアクション |
| `Cmd+[` | ジャンプ履歴: 戻る（`gd` で飛んだ後に元の場所へ） |
| `Cmd+]` | ジャンプ履歴: 進む |
| `Space` → `[` | ジャンプ履歴: 戻る (同上) |
| `Space` → `]` | ジャンプ履歴: 進む (同上) |

※ `Cmd+P` / `Cmd+Shift+F` は Alacritty 経由で F13 / F14 に変換して動作。
※ `Cmd+[` / `Cmd+]` は F16 / F17 に変換。

| キー | 操作 |
|---|---|

※ Alacritty 経由で F20 に変換して動作。

## バッファ切り替え（タブ風）

bufferline.nvim により上部にバッファが番号付きタブバーとして表示される。

| キー | 操作 |
|---|---|
| `gt` | 次のバッファ |
| `gT` | 前のバッファ |
| `Space` → `1`〜`9` | 番号を指定してバッファに移動 |
| `Space` → `bd` | バッファを閉じる（タブを閉じる） |
| `Space` → `bD` | すべてのバッファを閉じて空のバッファだけ残す |
| `:q` | バッファを閉じる（最後の1つなら Neovim を終了） |
| `:qa` | Neovim を終了 |

`:q` は分割ウィンドウがある場合はウィンドウを閉じる。バッファが複数ある場合はバッファを閉じる。最後のバッファなら Neovim を終了。

## タブページ

| キー | 操作 |
|---|---|
| `Space` → `tn` | 次のタブページ |
| `Space` → `tp` | 前のタブページ |
| `:tabnew ファイル名` | 新しいタブページで開く |
| `:tabclose` | タブページを閉じる |

※ タブページはウィンドウレイアウトの入れ物。バッファ（ファイル）の切り替えには `gt` / `gT` を使う。

## ファイルエクスプローラ (Snacks explorer)

画面右側にツリー形式のファイルエクスプローラを表示する。隠しファイル・gitignore 対象ファイルもデフォルトで表示。

| キー | 操作 |
|---|---|
| `Space` → `e` | ファイルエクスプローラを開く/閉じる |
| `-` | ファイルエクスプローラを開く/閉じる (同上) |

## Git

### 差分・blame (gitsigns)

| キー | 操作 |
|---|---|
| `Space` → `gd` | 前回コミットとの差分を左右分割で表示 (トグル) |
| `Space` → `gb` | カーソル行の最終変更者・コミットを表示 (blame) |
| `Space` → `gp` | カーソル位置の変更内容をポップアップ表示 |
| `]c` | 次の変更箇所へジャンプ |
| `[c` | 前の変更箇所へジャンプ |

※ 行番号の横に変更マークが自動表示される。

### コミット履歴ブラウズ (diffview)

| キー | 操作 |
|---|---|
| `Space` → `gh` | 現在ファイルのコミット履歴を開く |
| `Space` → `gH` | リポジトリ全体のコミット履歴を開く |
| `Space` → `gc` | Diffview を閉じる |

#### Diffview 内の操作

| キー | 操作 |
|---|---|
| `j` / `k` | コミット一覧を上下移動 |
| `Enter` | 選択したコミットの差分を左右分割で表示 |
| `]x` | 次のコミットの差分へ |
| `[x` | 前のコミットの差分へ |
| `q` | Diffview を閉じる |

### ワークツリー管理 (git-worktree.nvim)

Neovim 内から git worktree の一覧表示・切替・作成・削除ができる。Telescope と連携。
ワークツリー切替時にセッションが自動保存・復元される。

| キー | 操作 |
|---|---|
| `Space` → `wt` | ワークツリー一覧を表示・切替 |
| `Space` → `wc` | 新しいワークツリーを作成 |

#### ワークツリー一覧 (Telescope) 内の操作

| キー | 操作 |
|---|---|
| `Enter` | 選択したワークツリーに切替 (セッション自動保存→復元) |
| `Ctrl+d` | ワークツリーを削除 |
| `Ctrl+f` | 強制削除のトグル |

#### ワークツリー作成時の入力

1. ワークツリーのパスを入力 (例: `../feat-login`)
2. ブランチ名を入力 (例: `feat/login`、空欄で新しいブランチを自動作成)

### LazyGit

#### LazyGit 内の操作

| `P` | プル |

## セッション復元 (persistence)

- **自動保存**: Neovim 終了時（`:qa` 等）にセッションが自動保存される。
- **自動復元**: `nvim` とだけ打って起動すると、前回そのディレクトリで開いていたバッファやウィンドウの状態が自動復元される。ファイル指定で起動した場合（`nvim somefile.ts`）は復元しない。
- **ワークツリー連携**: ワークツリー切替時にセッションが自動で保存・復元される。

| キー | 操作 |
|---|---|
| `Space` → `sr` | 現在のディレクトリのセッションを手動復元 |
| `Space` → `sl` | 最後に使ったセッションを復元 |
| `Space` → `sd` | セッション自動保存を一時停止 |

## テキストオブジェクト (treesitter-textobjects)

Treesitter の構文解析を利用して、関数・クラス・引数などを正確に選択・操作できる。
`d`, `c`, `y`, `v` と組み合わせて使う。

### 選択 (select)

| キー | 対象 |
|---|---|
| `af` | 関数まるごと（シグネチャ含む） |
| `if` | 関数の中身だけ |
| `ac` | クラス/型まるごと |
| `ic` | クラス/型の中身 |
| `aa` | 引数1個（カンマ含む） |
| `ia` | 引数1個（カンマなし） |
| `ai` | if文まるごと |
| `ii` | if文の中身 |
| `al` | ループまるごと |
| `il` | ループの中身 |

例: `daf` = 関数削除、`cif` = 関数の中身を書き換え、`via` = 引数を選択

### ジャンプ (move)

| キー | 操作 |
|---|---|
| `]f` | 次の関数の先頭へ |
| `[f` | 前の関数の先頭へ |
| `]a` | 次の引数へ |
| `[a` | 前の引数へ |

### 引数スワップ (swap)

| キー | 操作 |
|---|---|
| `Space` → `a` | カーソル下の引数を右隣と入れ替え |
| `Space` → `A` | カーソル下の引数を左隣と入れ替え |

## フォーマット (conform.nvim)

保存時に自動フォーマットが実行される。

| 言語 | フォーマッタ |
|---|---|
| Go | gofmt |
| JS / TS / JSX / TSX / JSON / JSONC / CSS / HTML / YAML / Markdown | prettierd → prettier |

## 診断表示 (LSP)

エラー・警告がエディタ内に下線とテキストで表示される。

| 種類 | 下線の色 |
|---|---|
| エラー | 赤 |
| 警告 | 黄色 |
| 情報 | 青 |
| ヒント | グレー |

| キー | 操作 |
|---|---|
| `Space` → `d` | カーソル行のエラー内容をポップアップ表示 |
| `]d` | 次のエラー・警告へジャンプ |
| `[d` | 前のエラー・警告へジャンプ |

## コード補完 (nvim-cmp)

入力中にプロパティ名・メソッド・変数などの補完候補が自動表示される。

| キー | 操作 |
|---|---|
| `Ctrl+Space` | 手動で補完候補を表示 |
| `Ctrl+e` | 補完を閉じる |
| `Enter` | 候補を確定 |
| `Ctrl+b` / `Ctrl+f` | ドキュメントをスクロール |

### LSP の操作

| コマンド | 操作 |
|---|---|
| `:LspRestart` | Language Server を再起動 |
| `:LspInfo` | 接続中の Language Server を確認 |

## JSDoc / TSDoc 生成 (neogen)

| キー | 操作 |
|---|---|
| `Space` → `jd` | カーソル位置の関数/クラスに対してドキュメントコメントを自動生成 |

JS は JSDoc 形式、TS は TSDoc 形式で出力される。

## ターミナル (Snacks terminal)

画面下部にターミナルを表示する。VSCode のターミナルパネルと同じ使い方。

| キー | 操作 |
|---|---|
| `Cmd+J` | ターミナルの表示 / 非表示を切替 |
| `Ctrl+w` → `k` | ターミナルからエディタに戻る |
| `Ctrl+w` → `j` | エディタからターミナルに移動 |

※ ターミナル内では通常のシェル操作が可能。Alacritty 経由で F19 に変換して動作。

## ステータスライン (lualine)

画面下部に常時表示。ワークツリーごとに異なるディレクトリとブランチが表示されるため、複数の Neovim インスタンスを区別できる。

| 位置 | 表示内容 |
|---|---|
| 左 | モード (NORMAL/INSERT 等) |
| 左中 | Git ブランチ名 |
| 中央 | ワークスペースのディレクトリパス + ファイル名 |
| 右 | ファイルタイプ、位置 |

## 画面分割

### 分割の作成

| コマンド | 操作 |
|---|---|
| `:vs ファイル名` | 縦分割 (**v**ertical **s**plit、左右に並べる) |
| `:hs ファイル名` | 横分割 (**h**orizontal **s**plit、上下に並べる) |
| `:vs` / `:hs` | ファイル名なしで現在のバッファを分割 |
| `Ctrl+w` → `v` | 縦分割（コマンドなし） |
| `Ctrl+w` → `s` | 横分割（コマンドなし） |

### 分割間の移動

| キー | 操作 |
|---|---|
| `Ctrl+w` → `h` | 左のウィンドウへ |
| `Ctrl+w` → `j` | 下のウィンドウへ |
| `Ctrl+w` → `k` | 上のウィンドウへ |
| `Ctrl+w` → `l` | 右のウィンドウへ |

### サイズ変更

| キー | 操作 |
|---|---|
| `Ctrl+w` → `=` | すべてのウィンドウを均等サイズにする |
| `Ctrl+w` → `>` / `<` | 幅を増やす / 減らす |
| `Ctrl+w` → `+` / `-` | 高さを増やす / 減らす |
| `10 Ctrl+w` → `>` | 幅を10単位で増やす（数字で量を指定） |

### ウィンドウの閉じ方・その他

| キー | 操作 |
|---|---|
| `:q` | 現在のウィンドウを閉じる |
| `Ctrl+w` → `o` | 現在のウィンドウ以外をすべて閉じる |
| `Ctrl+w` → `r` | ウィンドウの位置をローテーション |
| `Ctrl+w` → `T` | 現在のウィンドウを新しいタブページに移動 |

## コードジャンプ (LSP)

Nixvim により以下の LSP サーバーが自動設定される。プロジェクトのディレクトリで Neovim を開けば、`gd` で別ファイルの定義にジャンプ可能。

| 言語 | LSP サーバー |
|---|---|
| TypeScript | `ts_ls` |
| Go | `gopls` |
